# AES-My-File

使用 AES-256-GCM 算法加密和解密你的文件。

## 功能特性

- 🔐 使用 AES-256-GCM 加密算法，提供强大的安全性
- 📁 支持通过系统原生对话框选择文件
- 🔑 基于 PBKDF2 的密码派生（使用 HMAC-SHA1）
- 🎯 支持大文件分块加密/解密
- 🖥️ 跨平台支持：Windows、macOS、Linux

## 系统要求

- [Zig](https://ziglang.org/) 0.11.0

## 构建和运行

```bash
zig build run
```

## 构建可执行文件

```bash
zig build
```

生成的可执行文件位于 `zig-out/bin/aes-my-file`

## 使用方法

1. 运行程序后，会弹出系统文件选择对话框
2. 选择要加密或解密的文件：
   - 普通文件：选择后将进行加密操作，生成 `.enc` 文件
   - `.enc` 文件：选择后将进行解密操作，生成 `.dec` 文件
3. 输入密码（至少 5 个字符）
4. 加密时需要确认密码
5. 等待处理完成

## 技术细节

### 加密算法
- **加密方式**: AES-256-GCM (Galois/Counter Mode)
- **密钥派生**: PBKDF2-HMAC-SHA256，600,000 次迭代（OWASP 推荐）
- **盐值**: 每个文件使用独立的 16 字节随机盐值
- **密钥长度**: 256 位
- **随机数**: 每个加密块使用独立的随机 nonce

### 文件格式
加密后的文件格式：
```
[16 字节 Salt][块1: 16字节Tag + 12字节Nonce + 密文][块2: Tag + Nonce + 密文]...
```


### 文件对话框
使用 [Native File Dialog](https://github.com/mlabbe/nativefiledialog) 库，提供原生系统文件选择体验。

## 注意事项

⚠️ **安全建议**:
- 请使用强密码（建议至少 12 个字符，包含大小写字母、数字和特殊字符）
- 妥善保管你的密码，密码丢失将无法解密文件
- 加密的文件建议在解密成功并验证后再删除原文件
